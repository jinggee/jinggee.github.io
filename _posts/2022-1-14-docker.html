前端看得懂的Docker

jingge 1/14/2022 docker

在讲Docker之前，我们先看一下以前是怎么发布部署我们的应用程序的。

通常我们需要先在服务器上安装ngix、node、mysql等应用程序运行所依赖的环境、之后再对他们进行配置（包括创建用户、配置参数等等），然后再将前端的构建生成的静态文件上传到服务器上。如果我们有多套环境（开发、测试、生产），那么每个环境上都要安装调试一遍这些基础应用，这些琐碎的操作极大的降低了用户的工作效率。

# Docker与虚拟机的相比的优势

  • Docker容器很快，启动和停止可以在妙级实现，而虚拟机则动辄几分钟；
  • Docker对系统资源的需求更少
  • 通过类似 Git 设计理念的操作来方便用户获取、分发和更新应用镜像，==存储复用，增量更新==；
  • Docker 通过 Dockerfile ==支持灵活的自动化创建和部署机制==，以提高工作效率，并标准化流程。

传统虚拟机运行N个不同的应用就要启动N个虚拟机，每个虚拟机都分配了独占的内存，磁盘，cpu。而Docker只需要启 动N个很薄的容器，并将应用放到容器内，应用获得的是接近原生的性能。

# 核心概念

Docker有三个核心概念：

# 镜像

# 容器

# 仓库

# 镜像相关的常用操作

  • 拉取一个镜像
语法： docker [image] pull [options] NAME[:TAG]

docker pull ubuntu:18.04
// 获取一个ubuntu 18.04版本的基础镜像

如果不加tag，则默认选择latest标签，即获取最新的镜像

// 从 Docker Hub Ubuntu 仓库下载一个最新版本的 Ubuntu 操作系统的镜像：
docker pull ubuntu

严格的讲Name应该添加镜像仓库地址作为前缀，如果不加的话默认使用官方的Docker Hub服务

docker pull ubuntu:18.04

// 相当于

docker pull registry.hub.docker.com/ubuntu:18.04

如果从非官方的仓库下载，则需要在仓库名称前指定完整的仓库地址。

docker pull  registry.cn-beijing.aliyuncs.com/hyhy-aibox/ubuntu:18.04

docker pull 子命令支持的选项：

--all-tags, -a = true|false, 是否获取仓库中的所有镜像，默认为否 -－ disable-content-trust, 跳过镜像的内容校验，默认true

  • 查看本地主机上已有的镜像
docker image ls  或者 docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
myubuntu 1.0 93f41e1e6297 About an hour ago 56.6MB
ubuntu 18.04 df8edc186894 6 days ago 56.6MB
nginx latest eeb9db34b331 2 weeks ago 134MB
  • 使用tag命令添加镜像标签
# 添加一个新的mynginx:1.0标签
docker tag nginx:latest mynginx:1.0

查看当前镜像, 本地镜像多了一个mynginx, tag是1.0， 注意imageid和ngix镜像的imageid相同

REPOSITORY TAG IMAGE ID CREATED SIZE
mynginx 1.0 eeb9db34b331 2 weeks ago 134MB
nginx latest eeb9db34b331 2 weeks ago 134MB
  • 搜索镜像

docker search子命令可以用来搜索官方Docker hub仓库中的镜像，语法为：

docker search [option] keyword

支持的选项： --limit int, 限制输出结果的个数，默认25个 --format ring: 格式化输出内容； -f, --filter filter 过滤输出的内容

例如：

docker search --filter is-official=true nginx

# 搜索官方的nginx镜像
  • 删除镜像
docker rmi IMAGE 或者 docker image rm IMAGE

其中IMAGE可以是标签，或者镜像ID，例如：

# 根据镜像标签删除镜像

docker rmi ubuntu:latest

# 根据镜像id删除镜像

docker image rm 9f4877540c73

支持的选项：

-f 强制删除，如果镜像创建的容器在运行时，镜像文件默认是无法删除的，通过-f参数可以强制删除。

-no-prune: 不要清理未带标签的父镜像。

  • 清理镜像

使用Docker一段事件后，系统中可能会残留一些临时的镜像文件，以及一些没有被使用的镜像，可以通过docker image prune命令清理

docker image prune

支持的选项：

-a 删除所有无用的镜像

-f 强制清理

-filter filter 只清理符合指定过滤器的镜像

  • 创建一个镜像

创建镜像有三种方式：基于已有镜像的容器创建、基于本地模板导入、基于dockerfile创建

  1. 基于已有的镜像容器创建
docker [container] commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]

CONTAINER 为容器ID 支持的选项有：

-a,--author="jingge" 作者信息 -m,--message="1212" 提交信息

举例：先启动一个镜像，并在其中修改添加一个test文件

docker run -it ubuntu:18.04 /bin/bash

tash test

exit

查看当前容器，id为a0631b9a1134

docker ps -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
a0631b9a1134 ubuntu:18.04 "/bin/bash" 3 minutes ago Exited (0) 3 minutes ago determined_shirley

运行下面命令，基于当前容器创建一个新的镜像

docker commit -a="jingge" -m="新增一个文件" a0631b9a1134 myubuntu:1.0

查看本机上的镜像，已经有了我们刚才创建的镜像

docker ps -a
myubuntu 1.0 93f41e1e6297 6 seconds ago 56.6MB
ubuntu 18.04 df8edc186894 6 days ago 56.6MB
  1. 基于dockerfile创建镜像 创建一个Dockerfile文件
FROM nginx
RUN ...

构建镜像

docker build -t mynginx:1.0  .

# 注意最后的 . 代表本次执行的上下文路径
  • 镜像的导出和载入
docker [image] save [options]

# 将镜像导出为本地文件

支持的选项有：-o, -output, 举例：

docker image save -o mynginx.tar nginx:latest

# 将最新的nginx镜像导出到本地文件mynginx.tar

之后，用户就可以通过复制 mynginx.tar文件将该镜像分享给他人。

镜像的载入：

docker [image] load [options]

选项有：-i或-input, 例如导入刚才的镜像文件

docker load -i mynginx.tar

或者

docker load< mynginx.tar
